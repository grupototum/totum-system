## 2. Server Compatibility Analysis

The following hardware specifications were provided for the dedicated server:

| Component | Specification | AI Generative Evaluation | Observations |
|---|---|---|---|
| CPU | Intel i5-2400 (4C/4T) 3.1GHz | ⚠️ Weak - Sandy Bridge from 2011 | While OpenClaw itself is a Node.js application and relatively lightweight, this CPU is considered weak for general AI tasks. For running local AI models or heavy automation, a more modern and powerful CPU would be highly recommended [3]. |
| RAM | 8GB DDR3 | ❌ Insufficient - minimum 32GB | OpenClaw requires Node.js 22+ and runs on macOS, Linux, or Windows via WSL2. For most users, a modern laptop is enough unless you run local models or heavy automation. The official documentation recommends 16GB or more for heavier workloads. For running multiple OpenClaw instances or 32B/70B parameter models, at least 32GB of RAM is required [2, 5]. Your 8GB of RAM will likely be a significant bottleneck for anything beyond very light usage. |
| GPU | Intel HD Graphics (integrated) | ❌ Useless for generative AI | OpenClaw does not utilize a GPU for its core operations, as it is a Node.js application. Therefore, the lack of a dedicated GPU is not a direct impediment to running OpenClaw itself [4]. However, if you plan to integrate with AI models that leverage GPU acceleration, this integrated GPU will not be sufficient. |
| Storage | SSD 240GB + HD 2TB | ⚠️ HDD can be reused, SSD small | OpenClaw requires 10-20GB of free space, with SSD strongly recommended for faster logs, cache access, and model downloads [3, 2]. The 240GB SSD is sufficient for the OpenClaw installation and its operational files. The 2TB HDD can be used for general data storage. |

**Conclusão da Compatibilidade:**

O servidor possui um **CPU** fraco e uma quantidade de **RAM** que é provavelmente insuficiente para a maioria dos casos de uso do OpenClaw, especialmente se houver a intenção de rodar modelos de IA localmente ou realizar automações pesadas. O **armazenamento** é adequado, e o **sistema operacional** (Linux) é o recomendado. É importante estar ciente das limitações de desempenho que o CPU e a RAM podem impor.

## 3. Pré-requisitos

Antes de instalar o OpenClaw, certifique-se de que os seguintes pré-requisitos estejam atendidos no seu servidor Linux:

*   **Sistema Operacional:** Linux (Ubuntu 22.04 já está presente, o que é ideal).
*   **Node.js:** Versão 22 ou superior. Você precisará instalar ou atualizar o Node.js para a versão mais recente. É a causa mais comum de falhas de instalação [1].
*   **curl:** Geralmente pré-instalado na maioria das distribuições Linux. É necessário para baixar o script de instalação.
*   **npm:** Vem junto com a instalação do Node.js.

## 4. Passos de Instalação do OpenClaw

Siga os passos abaixo para instalar o OpenClaw no seu servidor Linux:

### Passo 4.1: Atualizar o Sistema

É sempre uma boa prática garantir que seu sistema esteja atualizado:

```bash
sudo apt update && sudo apt upgrade -y
```

### Passo 4.2: Instalar ou Atualizar Node.js (versão 22+)

Se você não tiver o Node.js 22+ instalado, ou se tiver uma versão mais antiga, siga estas instruções. Recomenda-se usar o `nvm` (Node Version Manager) para gerenciar as versões do Node.js.

1.  **Instalar nvm (se não tiver):**

    ```bash
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash
    ```

    Após a instalação, feche e reabra o terminal, ou execute `source ~/.bashrc` (ou `~/.zshrc`, dependendo do seu shell) para carregar o `nvm`.

2.  **Instalar Node.js 22:**

    ```bash
    nvm install 22
    nvm use 22
    nvm alias default 22
    ```

    Verifique a versão instalada:

    ```bash
    node -v
    npm -v
    ```

### Passo 4.3: Instalar o OpenClaw

Com o Node.js configurado, você pode instalar o OpenClaw usando o script oficial:

```bash
curl -fsSL https://openclaw.bot/install.sh | bash
openclaw onboard --install-daemon
```

O primeiro comando baixa e executa o script de instalação, que instalará a CLI do OpenClaw. O segundo comando inicia o processo de *onboarding*, que o guiará pela configuração inicial, incluindo a conexão com canais de mensagens e provedores de modelos de IA [1]. A flag `--install-daemon` configura o OpenClaw para rodar como um serviço em segundo plano.

## 5. Configuração Inicial e Verificação

Após a instalação, o comando `openclaw onboard` o guiará através de um assistente para configurar o OpenClaw. Você precisará:

*   Emparelhar um canal de mensagens (ex: WhatsApp, Telegram).
*   Selecionar um provedor de modelo de IA (ex: OpenAI, Gemini).
*   Habilitar habilidades (skills) para o OpenClaw.

Para verificar a instalação e as dependências, você pode usar os seguintes comandos:

```bash
openclaw doctor
openclaw status
openclaw health
```

É recomendável executar uma pequena tarefa de teste após o *onboarding* para confirmar que tudo está funcionando corretamente [1].

## 6. Considerações Finais

Devido às especificações de hardware do seu servidor, especialmente o CPU e a RAM, é provável que o desempenho do OpenClaw seja limitado para tarefas que exigem muitos recursos, como a execução de modelos de IA localmente ou automações complexas e múltiplas instâncias. Para um desempenho ideal e para explorar todo o potencial do OpenClaw, especialmente com modelos de IA mais avançados, um upgrade de hardware (CPU e RAM) seria altamente benéfico.

Lembre-se de seguir as **melhores práticas de segurança**, como não expor *endpoints* de administração à internet pública e usar acesso com privilégios mínimos [1].

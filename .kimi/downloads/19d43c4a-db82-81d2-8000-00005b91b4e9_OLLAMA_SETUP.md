## Como usar

### Teste rápido
```bash
curl -X POST http://localhost:11434/api/generate -d '{
  "model": "llama3.2",
  "prompt": "Sua pergunta aqui",
  "stream": false
}'
```

### Integração com OpenClaw/n8n
```javascript
// Exemplo de chamada via HTTP Request
const response = await fetch('http://localhost:11434/api/generate', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    model: 'llama3.2',
    prompt: prompt,
    stream: false,
    options: {
      temperature: 0.7,
      num_predict: 500
    }
  })
});

const data = await response.json();
console.log(data.response);
```

## Quando usar Llama vs Kimi Cloud

| Tarefa | Modelo | Motivo |
|--------|--------|--------|
| Classificar mensagens | Llama | Rápido, barato |
| Responder FAQs simples | Llama | O(1) latency |
| Gerar copy básica | Llama | Custo zero |
| Estratégia criativa | Kimi | Qualidade superior |
| Análise complexa | Kimi | Raciocínio profundo |
| Validação importante | Kimi | Confiabilidade |

## Comandos úteis

```bash
# Ver modelos instalados
ollama list

# Baixar outro modelo
ollama pull llama3.1

# Remover modelo
ollama rm llama3.2

# Ver logs do servidor
journalctl -u ollama -f

# Parar servidor
pkill ollama

# Iniciar servidor
ollama serve
```

## Uso de recursos
- **RAM necessária:** ~4GB para llama3.2
- **CPU:** Funciona bem em CPU (não precisa de GPU)
- **Latência:** ~2-5s para respostas curtas

## Configuração no OpenClaw

Adicionar ao `config.yaml` do gateway:
```yaml
models:
  local-llama:
    provider: ollama
    endpoint: http://localhost:11434
    model: llama3.2
```

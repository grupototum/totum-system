## 🎛️ Sistema de Tarefas + Agentes - Totum Apps

### ⏰ Timestamp
- **Horário:** ~20:58 CST (Shanghai) / ~09:58 BRT (Israel)

### Resumo
Implementação completa do sistema de tarefas com visualização Gantt e integração de agentes no Apps Totum. Quatro componentes principais entregues.

### Componentes Criados

#### 1. useTasks.ts (atualizado)
- Novos campos: `milestone_id`, `data_inicio`, `data_fim`, `progresso`, `dependencias`
- Campos para agentes: `agente_id`, `recorrencia`, `horario_execucao`, `params`
- Funções novas: `atualizarProgresso()`, `atualizarDatas()`, `criarMilestone()`, `getTarefasPorMilestone()`, `getTarefasAgente()`

#### 2. GanttChart.tsx (novo)
- Visualização em timeline com 3 modos: Dia (14 dias), Semana (42 dias), Mês (90 dias)
- Barras coloridas por responsável (Israel #3B82F6, Pablo #F97316, Data #8B5CF6, Hug #10B981, TOT #6B7280)
- Progresso visual na barra
- Linha "Hoje" destacada
- Drag-and-drop para alterar datas

#### 3. useAgentTasks.ts (novo)
- Hook completo para gerenciar tarefas de agentes
- Agendamento automático com recorrência (única, diária, semanal, mensal)
- Execução manual ou automática por horário
- Scheduler verifica a cada 1 minuto tarefas pendentes
- Logs de execução com status (sucesso/erro/executando)
- Simulação de execução (placeholder para integração real)

#### 4. AgentTaskManager.tsx (novo)
- Dashboard com estatísticas por agente
- Cards dos 4 agentes: Pablo (Operações), Data (Tech), Hug (Radar), TOT (Orquestrador)
- Filtro por agente
- Formulário completo de criação/edição
- Lista de tarefas com accordion para detalhes
- Indicador de "Atrasada" quando próxima_execucao < agora

#### 5. ActionPlan.tsx (atualizado)
- 4 abas: Fases | Gantt | Tarefas | Agentes
- Integração com todos os hooks
- Modal de tarefa unificado

### Migração SQL Criada
Arquivo: `/apps/totum/migrations/008_logs_agentes.sql`
- Tabela `logs_execucao_agente` para armazenar execuções
- Índices: tarefa_id, agente_id, status, iniciado_em
- RLS policies para SELECT/INSERT
- Realtime enabled
- Colunas adicionais em `tarefas`: proxima_execucao, ultima_execucao, ultimo_resultado
